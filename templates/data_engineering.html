Data Engineering

Limpieza, transformación, encoding, feature engineering y normalización del dataset de 1.200 BIC para maximizar el potencial predictivo del modelo.

01

Limpieza de Datos

Se aplicaron reglas lógicas estrictas para garantizar la integridad del dataset antes del modelado.

Regla 01
Corrección de valores negativos en PM10
Se detectaron valores negativos en PM10_Promedio_ugm3 (mín: -13.5 μg/m³), físicamente imposibles. Se reemplazan con el valor mínimo válido de 0.
# Regla 1: PM10 no puede ser negativo df['PM10_Promedio_ugm3'] = df['PM10_Promedio_ugm3'].clip(lower=0) # Verificación print(df['PM10_Promedio_ugm3'].min()) # → 0.0
Regla 02
Validación de rango de edad estructural
Se valida que todas las estructuras tengan entre 50 y 450 años, rango históricamente plausible para BIC colombianos. Se detectaron 3 outliers extremos.
# Regla 2: Edad entre 50 y 450 años df = df[df['Edad_Estructura_Anios'].between(50, 450)] print(f"Registros válidos: {len(df)}") # → 1197
Regla 03
Verificación de ausencia de valores nulos
Se confirma que el dataset no contiene valores NaN en ninguna columna. Resultado: 0 nulos en las 11 variables.
# Regla 3: Sin valores nulos print(df.isnull().sum()) # → todos 0 assert df.isnull().sum().sum() == 0, "Error: hay valores nulos"
Regla 04
Validación de humedad relativa ≤ 100%
Se verifica que la humedad relativa no supere el 100%, límite físico absoluto. El dataset cumple esta condición (máx: 99.4%).
# Regla 4: Humedad ≤ 100% assert df['Humedad_Prom_Anual_pct'].max() <= 100 print(f"Humedad máx: {df['Humedad_Prom_Anual_pct'].max()}") # → 99.4

02

Codificación de Variables Categóricas

Los algoritmos de ML requieren entradas numéricas. Se aplica One-Hot Encoding a Material_Principal y Ciudad, generando 16 columnas binarias nuevas.

# One-Hot Encoding — Material_Principal y Ciudad from sklearn.preprocessing import OneHotEncoder import pandas as pd df_encoded = pd.get_dummies( df, columns=['Material_Principal', 'Ciudad'], prefix=['mat', 'city'], drop_first=False ) print(f"Columnas antes: 11 → después: {df_encoded.shape[1]}") # → 27

03

Feature Engineering

Se crean 2 nuevas variables que capturan riesgos sinérgicos no representados por las variables individuales.

Nueva variable 01
Índice de Lluvia Ácida
Combina precipitación anual y PM10 para estimar el impacto de la lluvia ácida sobre los materiales de construcción. A mayor precipitación y mayor contaminación, mayor riesgo de erosión química.
# Feature 1: Índice de Lluvia Ácida df['Indice_Lluvia_Acida'] = ( df['Precipitacion_Anual_mm'] * df['PM10_Promedio_ugm3'] ) / 1000 # Media: 40.61 | Std: 31.10 | Rango: 0 – 153.07
Nueva variable 02
Índice de Riesgo por Vibración
Relaciona la edad de la estructura con su proximidad a vías de alto tráfico. Estructuras más antiguas y más cercanas a vías tienen mayor susceptibilidad al daño por vibración mecánica.
# Feature 2: Índice de Riesgo por Vibración df['Riesgo_Vibracion'] = ( df['Edad_Estructura_Anios'] / df['Distancia_Via_Principal_m'] ).round(3) # Media: 7.13 | Std: 16.79 | Rango: 0.07 – 154.0

05

Normalización y Escalamiento

Las variables numéricas tienen rangos muy distintos (2m a 672m vs. 51% a 99%). Se aplica Min-Max Scaler para llevar todas al rango [0, 1].

# Normalización Min-Max — variables numéricas from sklearn.preprocessing import MinMaxScaler numeric_cols = [ 'Edad_Estructura_Anios', 'Precipitacion_Anual_mm', 'Humedad_Prom_Anual_pct', 'PM10_Promedio_ugm3', 'Distancia_Via_Principal_m', 'Indice_Lluvia_Acida', 'Riesgo_Vibracion' ] scaler = MinMaxScaler() df[numeric_cols] = scaler.fit_transform(df[numeric_cols]) # Resultado: todas las variables en rango [0.0, 1.0]
